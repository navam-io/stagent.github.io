# Human-in-the-Loop

The Inbox is the agent-to-human communication hub ŌĆö an email-style interface where permission requests, agent questions, task completions, and failures surface for human review and action.

<BrowserFrame src="/screenshots/inbox-notifications.png" alt="Inbox showing agent permission requests awaiting approval" />

## Key Concepts

- **Permission Requests** ŌĆö When an agent wants to use a tool (read a file, execute code, send data), a notification appears in the Inbox with full context. Click Allow or Deny
- **Agent Questions** ŌĆö The `AskUserQuestion` tool surfaces clarifying questions as conversational messages. Respond directly in the inbox
- **Task Completions** ŌĆö Get notified when agents finish work, with links to results
- **Task Failures** ŌĆö Error notifications with full context for debugging

## Notification Types

| Type | Action Required | Description |
|------|:---:|-------------|
| `permission_required` | Yes | Agent wants to use a tool ŌĆö review and decide |
| `agent_message` | Yes | Agent is asking a clarifying question |
| `task_completed` | No | Informational ŌĆö task finished successfully |
| `task_failed` | No | Informational ŌĆö task encountered an error |

## How It Works

The notification table acts as a message queue. When `canUseTool` fires, a row is written to the `notifications` table with the tool name, parameters, and context. The agent enters a polling loop, checking for a response every few seconds.

When the user clicks Allow or Deny, their response is written via `POST /api/tasks/[id]/respond`. The polling loop detects the response and returns it to the Agent SDK.

## Use Cases

### Compliance-First Oversight

**Product Team** ┬Ę *Enforcing access policies across agent operations*

Your compliance officer reviews every agent tool call. Read operations on internal docs get quick Allow clicks, but any write to production systems gets a Deny with a note explaining why. The notification history becomes an audit trail ŌĆö every decision logged with timestamp, reviewer, and rationale.

### Interactive Research Session

**Knowledge Worker** ┬Ę *Collaborating with an agent on a financial analysis*

The agent encounters conflicting revenue figures in two uploaded reports. Instead of guessing, it surfaces a clarifying question via `AskUserQuestion`: "Q3 revenue is $4.2M in the annual report but $3.8M in the quarterly filing ŌĆö which source should I prioritize?" You respond directly in the Inbox, and the agent continues with the correct figure.

### Learning to Trust AI

**Non-Technical User** ┬Ę *Gradually reducing oversight as confidence grows*

You start with every tool call requiring approval ŌĆö 15 notifications for a simple document analysis task. But after watching the agent make safe, predictable choices across 50+ operations over a week, you begin trusting its judgment. The Inbox becomes a spot-check rather than a gatekeep, and your workflow accelerates dramatically.
