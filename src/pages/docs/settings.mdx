# Settings

The settings page is where operators define how Stagent authenticates, which tool requests can be trusted by default, and how the local workspace data should be managed.

<BrowserFrame src="/screenshots/settings-list.png" alt="Settings page showing authentication, tool permissions, and data management sections" />

## Key Concepts

- **Authentication Modes** — Connect Stagent with OAuth or a direct API key depending on your operating model.
- **Default Tool Permissions** — Seed safe approval patterns so the inbox does not need to mediate every low-risk action.
- **Data Management** — Reset the workspace when you need to clear projects, tasks, workflows, schedules, documents, logs, and notifications.
- **Local-First Configuration** — Settings are managed close to the workspace instead of hidden inside a remote SaaS control plane.

## Authentication Methods

| Method | Description | Best For |
|--------|-------------|----------|
| **OAuth** | Built-in sign-in flow for Claude subscriptions | Individual operators and local evaluation |
| **API Key** | Direct credential entry for Agent SDK connectivity | Development, testing, and controlled server setups |

## Data Management

- **Clear All Data** — Removes projects, tasks, workflows, schedules, documents, logs, notifications, and seeded profiles while preserving the underlying setup.
- **Permission Pattern Curation** — Operators can review and revoke auto-approved command patterns from one place.

## How It Works

Settings centralize the trust model for the rest of the product. The authentication section controls how runs are authorized, the permissions section shapes inbox volume, and the data tools make it possible to reset or repopulate the workspace safely.

The visible grouping also keeps operational choices legible. You do not have to remember where an approval rule lives or how the workspace is authenticated; both are explicit on one screen.

## Use Cases

### Team Onboarding Setup

**Product Team** · *Getting new team members started in under 5 minutes*

A new teammate connects with OAuth, reviews the default approval patterns, and resets the sample workspace before starting real work. Setup stays legible because the auth, permission, and data controls are grouped together.

### API Key Rotation

**Developer** · *Monthly credential rotation for security compliance*

Your security team rotates access monthly. You update the auth mode or credential, remove any stale auto-approval patterns, and keep the control surface for trust decisions in one place.

### Starting Fresh

**Non-Technical User** · *Resetting after initial experimentation*

You spent a week exploring seeded sample data. When it is time to move to production work, one reset clears the operational records while leaving the product ready for a clean start.
