# Project Management

Projects are the top-level organizational unit in Stagent. Each project contains tasks, documents, and configuration — everything needed for a complete body of work.

<BrowserFrame src="/screenshots/projects-list.png" alt="Projects list showing active projects with task counts" />

## Key Concepts

- **Project Cards** — Card-based overview showing status indicators, task counts, and descriptions
- **Project Detail** — Drill into any project for its full task board, document attachments, and settings
- **Status Tracking** — Projects track aggregate progress across all contained tasks
- **REST API** — Full CRUD via `/api/projects/[id]` for programmatic access

<BrowserFrame src="/screenshots/project-detail.png" alt="Project detail view showing task board and document attachments" />

## How It Works

Projects use Server Components for read operations — the project list page queries the database directly with Drizzle ORM. Mutations (create, update, delete) go through API routes for proper validation.

Each project has a unique ID generated via `crypto.randomUUID()`. Task counts are computed via left joins with the tasks table, grouped by status.

## Use Cases

### Multi-Sprint Product Launch

**Product Team** · *Coordinating a cross-functional release*

Your product launch has 30+ tasks spanning content creation, documentation updates, and QA testing. You create a single project, attach brand guidelines and API specs as documents, and organize tasks by sprint. The project card on the dashboard shows aggregate progress — 18 completed, 8 running, 4 planned — giving stakeholders a single number to track.

### Personal Finance Organization

**Non-Technical User** · *Preparing for tax season*

You create a "Tax Prep 2025" project and upload W-2s, 1099s, and mortgage interest statements. Each tax section becomes a task — wages, investments, deductions, credits. The agent processes documents against each task, extracting relevant figures and flagging missing forms. Everything lives in one place instead of scattered across folders.

### Client Engagement Tracking

**Knowledge Worker** · *Managing deliverables across multiple clients*

Each client gets their own project with tasks for active deliverables. You use the REST API (`/api/projects/[id]`) to pull project status into your weekly client report — task counts by status translate directly into progress percentages. When a client asks "where are we?", the answer is one API call away.
