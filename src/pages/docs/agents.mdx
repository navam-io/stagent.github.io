# Agent Integration

Stagent integrates with the Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`) to execute tasks autonomously while maintaining human oversight at every step.

<BrowserFrame src="/screenshots/monitor-logs.png" alt="Monitor showing real-time agent execution logs" />

## Key Concepts

- **Claude Agent SDK** — TypeScript SDK for running Claude Code as an embedded agent with streaming, tool use, and governance callbacks
- **canUseTool Callback** — Invoked every time the agent attempts to use a tool. Returns `allow` or `deny` for real-time human oversight
- **Fire-and-Forget Execution** — `POST /api/tasks/[id]/execute` returns HTTP 202 immediately; the agent runs asynchronously
- **Session Management** — Each execution creates a session with a unique `sessionId`. Failed or interrupted tasks can be resumed from where they left off
- **Agent Logs** — Every tool invocation, decision, and result is logged to the `agent_logs` table for complete auditability

## Execution Flow

1. User queues a task from the Kanban board
2. Execution manager creates an Agent SDK session
3. Agent processes the task, calling tools as needed
4. Each tool call triggers `canUseTool` → notification → human review
5. Agent continues based on human response (allow/deny)
6. Results and logs are persisted to the database

## How It Works

The execution manager (`src/lib/agents/execution-manager.ts`) wraps the Agent SDK with Stagent-specific concerns: task status updates, log persistence, notification creation, and error recovery.

The `canUseTool` callback writes a notification to the database. A polling loop in the agent checks for a human response. This uses the database as a message queue — simple, reliable, and fully auditable.

## Use Cases

### Governed Code Deployment

**Product Team** · *Maintaining deployment safety with human oversight*

Your agent completes a code refactor and attempts `git push`. The `canUseTool` callback fires, and a notification appears in your Inbox with the full diff. You review the changes, confirm they match the task requirements, and click Allow. The push goes through with a complete audit trail — who approved it, when, and what was pushed.

### Research with Source Verification

**Knowledge Worker** · *Ensuring agents use credible sources*

The agent attempts to fetch data from a paywalled research database. You see the permission request, deny access to the paywalled source, and provide an alternative open-access URL via the `AskUserQuestion` response. The agent adjusts its approach and continues with the verified source — no hallucinated citations, no unauthorized access.

### Batch Processing with Session Resume

**Developer** · *Refactoring 50 files without starting over*

Your agent is refactoring 50 files when it fails at file #23 due to an unexpected import pattern. The session is preserved with its full context. You fix the edge case in the source code, then resume the session from the checkpoint — the agent picks up at file #23 and completes the remaining 27 files without re-processing the first 22.

### First-Time Agent Delegation

**Non-Technical User** · *Building trust through full transparency*

Every tool call surfaces in your Inbox for review — reading a file, writing output, accessing the network. You see exactly what the agent wants to do before it does it. After approving 50+ benign operations and seeing consistent, safe behavior, you develop confidence in the agent's judgment and begin approving faster.
