# Real-time Monitoring

The monitoring dashboard provides real-time visibility into agent activity — showing active agents, task assignments, streaming logs, and an activity timeline.

<BrowserFrame src="/screenshots/monitor-logs.png" alt="Monitor view showing real-time agent execution logs with tool calls" />

## Key Concepts

- **Server-Sent Events (SSE)** — Real-time log streaming from the server to the browser. No WebSockets needed — SSE is simpler, more reliable, and works through proxies
- **Agent Logs** — Every tool invocation, reasoning step, and result is logged to the `agent_logs` table and streamed live
- **Task Filtering** — Filter logs by task, by agent session, or view all activity across the system
- **Activity Timeline** — Chronological view of all agent actions with timestamps and durations

## Log Entry Types

Each log entry includes:
- **Timestamp** — When the action occurred
- **Task ID** — Which task the agent is working on
- **Event Type** — Tool call, reasoning, result, error, or status change
- **Content** — The full detail of what happened

## How It Works

The monitoring endpoint (`/api/logs/stream`) uses a `ReadableStream` with a polling loop. The server checks the `agent_logs` table for new entries at regular intervals and pushes them to connected clients via SSE.

The browser receives these events and appends them to the log view in real time. Each entry is rendered with syntax highlighting for tool calls and code blocks.

## Use Cases

### Live Demo for Stakeholders

**Product Team** · *Showcasing agent capabilities to executives*

You project the monitoring dashboard on the conference room screen and kick off a code analysis task. The SSE logs stream in real time — executives watch the agent read files, identify patterns, and generate a summary. The live feed makes the agent's reasoning transparent and builds organizational buy-in for AI adoption.

### Debugging a Failed Task

**Developer** · *Tracing a failure to its root cause*

A nightly data-processing task failed. You filter the monitoring logs by task ID and scroll through the execution timeline. At step 5, the agent attempted an API call that returned a 401 — an expired token. The log shows the exact request, response, and the agent's error-handling attempt. You rotate the token and re-queue the task.

### Overnight Batch Monitoring

**Power User** · *Morning triage of unattended agent work*

You queued 10 tasks before leaving the office. In the morning, the monitoring dashboard shows the results at a glance: 8 completed, 1 failed (network timeout at 3 AM), 1 still running. You click into the failed task's log stream, see the exact failure point, and decide whether to retry or reassign.
