# Workflow Engine

The workflow engine chains tasks into automated multi-step workflows. Three patterns cover the most common enterprise agent use cases.

<BrowserFrame src="/screenshots/workflows-list.png" alt="Workflows list showing configured automation pipelines" />

## Workflow Patterns

### 1. Sequence (Prompt Chaining)

Tasks execute in strict order. Each step's output feeds the next. Gate checks between steps ensure quality before proceeding.

**Example**: Research → Draft → Legal Review → Publish

### 2. Planner → Executor (Orchestrator-Workers)

A planning agent breaks down a goal into steps. Executor agents handle each step independently. Results roll up to the planner for synthesis.

**Example**: "Analyze Q1 financials" → identify data sources → extract metrics → generate report

### 3. Human-in-the-Loop Checkpoint

Agent executes autonomously until it reaches a defined checkpoint, then pauses for human approval before continuing.

**Example**: Draft client email → **human approves** → send email → log to CRM

<BrowserFrame src="/screenshots/workflow-detail.png" alt="Workflow detail view showing step configuration and execution status" />

## Key Concepts

- **JSON Workflow Definitions** — Workflows are defined declaratively with steps, dependencies, and gate conditions
- **Composable Patterns** — A Sequence workflow can include Checkpoint steps. A Planner-Executor can require human approval at critical junctures
- **Status-Driven Transitions** — The workflow engine uses task status changes to trigger the next step
- **Checkpoint & Resume** — Workflows pause at checkpoints and resume when the human responds

## How It Works

Workflow definitions are stored in the `workflows` table as JSON. The workflow engine (`src/lib/workflows/engine.ts`) interprets these definitions and orchestrates task execution through the execution manager.

Each workflow step creates a task in the `tasks` table. Step transitions are triggered by status changes — when a task completes, the engine checks the workflow definition for the next step and creates it.

## Use Cases

### Code Review Pipeline

**Product Team** · *Automating the full review cycle with safety gates*

You define a Sequence workflow: pull latest changes → run static analysis → execute security scan → **checkpoint for tech lead review** → post review comments to the PR. The checkpoint ensures no automated comments go live without human eyes on the security scan results. One workflow replaces four manual handoffs.

### Investment Research Report

**Knowledge Worker** · *Using Planner-Executor for structured analysis*

You create a Planner-Executor workflow with the goal "Analyze NVIDIA Q4 earnings impact on portfolio." The planner agent breaks this into five executor steps: gather earnings data, compare against estimates, analyze guidance, assess sector impact, and draft a recommendation. Each executor runs independently, and the planner synthesizes results into a coherent report.

### Sprint Planning Automation

**Developer** · *Combining sequence steps with human checkpoints*

Your workflow parses the previous sprint's task data → calculates velocity metrics → drafts next sprint's task breakdown → **pauses for PM review**. The PM reviews the generated tasks, adjusts priorities, and approves. The workflow resumes and creates the tasks in the project board — turning a 2-hour planning meeting into a 10-minute review.

### Meal Planning for the Week

**Non-Technical User** · *Personal automation with a review step*

You set up a simple Sequence workflow: gather dietary preferences → search for recipes → generate a 7-day meal plan → compile a shopping list → **checkpoint for your review**. You scan the plan, swap out Tuesday's dinner, approve, and the final shopping list is ready. The checkpoint ensures the AI's suggestions match what you'll actually cook.
