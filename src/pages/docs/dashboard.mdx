# Dashboard & Homepage

The homepage provides a bird's-eye view of your entire Stagent workspace — surfacing key metrics, priority tasks, recent activity, and quick actions.

<BrowserFrame src="/screenshots/home-dashboard.png" alt="Stagent dashboard showing project overview with task statistics" />

## Key Concepts

- **Contextual Greeting** — Time-based salutation with live database counts (running tasks, awaiting review, failed tasks)
- **Project Cards** — Each project shows task counts by status, with quick-access links to the Kanban board
- **Recent Activity** — Real-time feed of the latest agent actions, task completions, and notifications
- **Quick Actions** — One-click buttons for common operations: create project, add task, view inbox

<BrowserFrame src="/screenshots/project-detail.png" alt="Project detail view accessible from dashboard project cards" />

## How It Works

The dashboard uses React Server Components to query the SQLite database directly — no API calls needed for reads. This means the page loads with real data immediately, no loading spinners.

Each project card aggregates task counts from the `tasks` table, grouped by status. Running task counts update on page refresh as agents complete work.

## Use Cases

### Morning Standup at a Glance

**Product Team** · *Team lead opening Stagent at 9 AM*

Your overnight agents processed pull requests, ran regression tests, and flagged two failed deployments. The dashboard greeting shows "3 running, 2 awaiting review, 1 failed" — you click through to the failed task, see the error context, and reassign it before the standup even starts.

### Personal Project Portfolio

**Non-Technical User** · *Managing three personal projects simultaneously*

You're tracking a home renovation, an investment portfolio rebalance, and a vacation itinerary. Each project card shows overnight activity — the renovation agent found three contractor quotes, the investment agent flagged a rebalancing opportunity, and the vacation agent completed flight comparisons. One glance, three projects, zero context switching.

### Sprint Health Check

**Developer** · *Checking task throughput mid-sprint*

You notice the Queued column count is climbing — five tasks are stuck waiting for permission approvals. A quick click through to the Inbox reveals they're all blocked on a `git push` permission wall. You batch-approve the safe ones and get the pipeline flowing again.
