# Task Management

The Kanban board is the primary interface for managing agent tasks. Tasks flow through columns representing execution states: **Planned → Queued → Running → Completed/Failed**.

<BrowserFrame src="/screenshots/kanban-board.png" alt="Kanban board with tasks organized across status columns" />

## Key Concepts

- **Status Columns** — Visual workflow columns mirror the task execution lifecycle. Drag-and-drop between Planned and Queued to control when agents pick up work
- **Task Cards** — Each card shows title, priority, complexity, assigned agent, and progress indicators
- **AI-Assisted Task Creation** — Describe a goal in natural language; Claude breaks it down into actionable subtasks with appropriate priority and complexity ratings
- **Side Panel Detail** — Click any task to see full details, execution logs, and status history without leaving the board

<BrowserFrame src="/screenshots/task-ai-assist.png" alt="AI-assisted task creation breaking down goals into subtasks" />

## Task Lifecycle

| Status | Description |
|--------|-------------|
| **Planned** | Task defined but not yet queued for execution |
| **Queued** | Ready for agent pickup |
| **Running** | Agent actively executing the task |
| **Completed** | Successfully finished |
| **Failed** | Agent encountered an error |
| **Cancelled** | Manually stopped by the user |

<BrowserFrame src="/screenshots/task-detail-panel.png" alt="Task detail panel showing full task information, status history, and execution logs" />

## How It Works

Tasks are stored in the `tasks` table with a status field governed by valid transitions defined in `task-status.ts`. The execution manager picks up queued tasks and creates Agent SDK sessions. Status transitions are atomic and logged.

The fire-and-forget pattern means `POST /api/tasks/[id]/execute` returns HTTP 202 immediately — the agent runs asynchronously while you continue working.

## Use Cases

### Code Review Pipeline

**Product Team** · *Processing a backlog of pull requests*

Twelve PRs landed overnight. You create a task for each, assign them to the code-reviewer agent profile, and drag the batch to Queued. The agent processes them sequentially — running static analysis, checking for security issues, and posting review comments. By the time your team logs on, every PR has detailed feedback waiting.

### Research Decomposition

**Knowledge Worker** · *Breaking down a complex analysis*

You type "competitive landscape analysis for Q2 product strategy" into the AI-assisted task creator. Claude breaks it into five subtasks: identify competitors, gather pricing data, analyze feature gaps, summarize market trends, and draft recommendations. Each subtask gets appropriate complexity ratings and lands in Planned, ready for you to prioritize.

### Daily Task Automation

**Power User** · *Recurring maintenance tasks every morning*

You keep a set of recurring tasks in Planned: lint the codebase, check for dependency updates, and generate a changelog. Each morning you drag them to Queued in order — the agent picks them up and runs them while you focus on feature work. Results appear in the side panel by the time you finish your coffee.

### Delegating Without Technical Knowledge

**Non-Technical User** · *Turning raw data into insights*

You attach a CSV of customer feedback to a new task and describe what you need: "Find the main themes and summarize sentiment." The AI-assist suggests a pipeline — data cleaning, sentiment analysis, theme extraction, and summary generation — each as a separate subtask with clear descriptions you can understand and approve.
