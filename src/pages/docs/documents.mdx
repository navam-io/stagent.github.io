# Document Management

The document manager provides a dedicated interface for uploading, organizing, and preprocessing documents that agents can use as context during task execution.

<BrowserFrame src="/screenshots/documents-table.png" alt="Documents table view showing uploaded files with metadata and status" />

<BrowserFrame src="/screenshots/document-upload-dialog.png" alt="Document upload dialog supporting multiple file formats" />

## Key Concepts

- **Multi-Format Upload** — Support for text, PDF, images, Office documents (Word, PowerPoint), and spreadsheets (Excel, CSV)
- **Text Extraction** — Automatic preprocessing extracts text content from uploaded files for agent consumption
- **Table & Grid Views** — Switch between detailed table view and visual grid view for browsing documents
- **Bulk Operations** — Select multiple documents for bulk delete or status updates
- **Agent Context Building** — Documents are automatically prepared as context for agent tasks

<BrowserFrame src="/screenshots/documents-grid.png" alt="Documents grid view showing visual card layout" />

## Document Pipeline

1. **Upload** — User uploads a file through the upload dialog
2. **Store** — File saved to `~/.stagent/uploads/` with metadata in the `documents` table
3. **Preprocess** — Text extraction runs for supported formats (PDF, Office, text)
4. **Index** — Extracted text stored in `extractedText` column for fast retrieval
5. **Serve** — Agent receives document context when executing related tasks

<BrowserFrame src="/screenshots/document-detail-pdf.png" alt="Document detail view showing PDF preview with extracted text and metadata" />

## Supported Formats

| Format | Processor | Extraction |
|--------|-----------|------------|
| Text (`.txt`, `.md`, `.csv`) | Direct read | Full text |
| PDF (`.pdf`) | pdf-parse v2 | Full text + metadata |
| Images (`.png`, `.jpg`) | image-size | Dimensions + metadata |
| Word (`.docx`) | mammoth | Full text |
| PowerPoint (`.pptx`) | jszip | Slide text |
| Excel (`.xlsx`) | xlsx | Cell data |

## Use Cases

### Onboarding Knowledge Base

**Product Team** · *Grounding agents in company-specific knowledge*

You upload your team's wiki pages, API documentation, and architecture diagrams. When an agent processes onboarding tasks, it draws from this document context — producing guides that reference your actual endpoints, your naming conventions, and your team's workflow. No generic boilerplate, just grounded, company-specific content.

### Contract Analysis

**Knowledge Worker** · *Comparing terms across multiple vendor agreements*

You upload 15 vendor contracts in PDF and DOCX formats. The text extraction pipeline processes each document, and the agent compares payment terms, SLA commitments, and auto-renewal clauses across all 15. The result is a structured comparison table with flagged risks — work that would take days done in minutes.

### Personal Document Organization

**Non-Technical User** · *Bringing order to a pile of paperwork*

You upload a mix of receipts, insurance documents, and warranty cards. The agent extracts dates, amounts, and vendor names from each document using the text extraction pipeline, then organizes everything into a spreadsheet with categories, expiration dates, and totals. Your filing cabinet becomes a searchable, structured database.
