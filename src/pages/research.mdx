## 1. Executive Summary

Stagent is the **coordination layer for the agentic enterprise** — an AI-powered task management platform where Claude agents execute tasks with human-in-the-loop oversight. In a market racing toward autonomous AI agents, Stagent occupies a critical gap: the governance and orchestration layer that makes enterprise adoption of agents safe, visible, and productive.

The thesis is simple. Every enterprise will deploy AI agents. Every agent will need a human who can see what it's doing, approve what it plans to do, and course-correct when it goes wrong. No serious enterprise will hand autonomous decision-making to an AI without oversight. Stagent provides that oversight through a UX paradigm every knowledge worker already understands — task boards, inboxes, and notification feeds. It is, in essence, **Jira for AI agents**.

Three value propositions define Stagent's market position:

1. **Familiar UX** — Kanban boards, project management, and notification inboxes map directly to how teams already manage work. The learning curve is near-zero.

2. **Local-first data sovereignty** — Stagent runs on your infrastructure with a local SQLite database. No data leaves your premises. In an era of tightening regulation (EU AI Act effective August 2026), this is not a feature — it is a compliance prerequisite.

3. **Native human-in-the-loop governance** — The Agent SDK's `canUseTool` callback is the central coordination mechanism. Every agent action can be intercepted, reviewed, and approved through a familiar inbox interface. Governance is the product, not a checkbox.

As of March 2026, Stagent has shipped **18 features across 7 development sprints** — from foundational database schema through agent integration, workflow orchestration, document management, and a full-featured browser-based UI.

## 4. Product Architecture

### Design Philosophy

Stagent's architecture follows three principles drawn from Anthropic's "Building Effective Agents" guide:

1. **Simplicity** — Start with simple agent designs; add complexity only when simpler solutions fall short. The database is SQLite, not PostgreSQL. The distribution is `npx stagent`, not a Kubernetes deployment.

2. **Transparency** — Show the agent's work to the user. The monitoring dashboard streams agent reasoning and tool calls in real time.

3. **Agent-Computer Interface (ACI)** — Stagent's `canUseTool` polling pattern provides a clean, reliable handoff mechanism between agent autonomy and human oversight.

### The canUseTool Governance Pattern

The centerpiece of Stagent's architecture is the `canUseTool` polling pattern — the mechanism that transforms autonomous agent execution into governed agent execution.

<Figure number={4} caption="canUseTool Governance Flow — 5-Step Human-in-the-Loop Cycle">
  <GovernanceFlow />
</Figure>

<Figure number={5} caption="Inbox Notifications — Human-in-the-Loop Approval Interface">
  <BrowserFrame src="/screenshots/inbox-notifications.png" alt="Inbox showing agent permission requests awaiting human approval" />
</Figure>

1. **Agent requests permission** — When the Claude agent wants to use a tool, the SDK calls the `canUseTool` callback.
2. **Notification created** — Stagent writes a notification record to the database.
3. **Human reviews in Inbox** — The notification appears in the user's Inbox — a familiar email-like interface.
4. **Human responds** — The user clicks Allow or Deny. Their response is written back to the database.
5. **Agent continues** — The polling loop detects the response and returns it to the Agent SDK.

This pattern uses the database as a message queue — a deliberately simple architecture that avoids the complexity of WebSockets or external message brokers while providing reliable, auditable human-agent coordination.

### Workflow Orchestration

Stagent ships three workflow patterns:

| Pattern | How It Works | Enterprise Example |
|---------|-------------|-------------------|
| **Sequence** | Tasks execute in strict order. Each step's output feeds the next. | Research → Draft → Legal Review → Publish |
| **Planner → Executor** | A planning agent breaks down a goal. Executors handle each step. | "Analyze Q1 financials" → extract → generate report |
| **HITL Checkpoint** | Agent executes until a checkpoint, then pauses for human approval. | Draft email → **human approves** → send → log to CRM |

### Positioning Matrix

The agent platform market can be mapped along two axes: **agent autonomy** and **human governance**. Most platforms optimize for one at the expense of the other.

Stagent occupies the **high-governance, high-autonomy** quadrant — the quadrant that enterprise buyers actually need.

### Three Core Differentiators

**1. Governance-First Architecture** — Security prevents unauthorized access. Governance ensures authorized actions are appropriate, visible, and auditable. Stagent's `canUseTool` pattern provides governance at the interaction level.

**2. Task Management UX** — Stagent maps to work management paradigms that billions of knowledge workers already understand. The Kanban board, the Inbox, the Monitor — all familiar patterns. No CLI, no developer console, no friction.

**3. Local-First Data Sovereignty** — Stagent's database lives at `~/.stagent/stagent.db` — a local SQLite file. No data transits to external servers. This is a regulatory strategy, not a philosophical preference.

## Appendix A: Technical Glossary

| Term | Definition |
|------|-----------|
| **Agent SDK** | Anthropic's Claude Agent SDK — a TypeScript library for running Claude Code as an embedded agent with programmatic tool permissions. |
| **canUseTool** | A callback invoked every time the agent attempts to use a tool. Returns `allow` or `deny` for real-time human oversight. |
| **Drizzle ORM** | TypeScript-first ORM for type-safe database queries with SQLite. |
| **Fire-and-forget** | API returns HTTP 202 immediately; agent runs asynchronously. |
| **HITL** | Human-in-the-loop — requiring explicit human approval at defined checkpoints. |
| **MCP** | Model Context Protocol — open standard for connecting AI to external tools. |
| **SSE** | Server-Sent Events — real-time streaming for the monitoring dashboard. |
| **SQLite WAL** | Write-Ahead Logging enabling concurrent reads during agent writes. |
